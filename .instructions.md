# 🌿 Follow The Aroma Recipe Upload Workflow

## When This Applies

User is asking about:
- Adding a new recipe to the blog
- Uploading recipe photos and content
- Publishing recipes to Sanity
- Creating multiple recipes at once
- Avoiding duplicate recipes

## Recommended Workflow

### For Single Recipe (Interactive)
**Use case**: Adding one recipe with photo at a time

```bash
node scripts/add-recipe-safe.mjs
```

**What it does**:
1. Prompts for recipe details (title, slug, description, cuisine)
2. Uploads photo directly (validates format and file exists)
3. Collects ingredients and steps
4. Shows summary for confirmation
5. Creates and publishes immediately
6. **Prevents duplicates** by checking slug against existing recipes

**When to use**: 
- ✅ Adding a single new recipe interactively
- ✅ When you have photos ready
- ✅ Preferred for most daily use cases

### For Bulk Upload (Batch)
**Use case**: Adding multiple recipes at once from JSON

1. **Prepare recipes file**:
   ```bash
   nano recipes-bulk.json
   ```
   Use format from RECIPE_WORKFLOW_GUIDE.md

2. **Upload photos**:
   ```bash
   node scripts/upload-photos.mjs
   ```

3. **Import recipes**:
   ```bash
   node scripts/bulk-import-recipes.mjs
   ```

**Note**: Bulk import creates directly as published (no draft step)

## Key Features

### Duplicate Prevention ✅
- Checks if recipe slug already exists
- Prevents creating duplicate entries
- Error if slug found: `Recipe with slug "xyz" already exists!`
- Solution: Use unique slug or delete existing recipe first

### Photo Handling ✅
- Integrated into workflow (no separate step)
- Supports: .jpg, .jpeg, .png, .webp
- Validates file exists before upload
- Returns photo ID automatically

### Direct Publishing ✅
- Recipes published immediately (not drafted)
- Available on site right after creation
- No separate publish step needed

## Field Requirements

| Field | Required | Notes |
|-------|----------|-------|
| title | ✅ | Recipe name (5-80 chars) |
| slug | ✅ | URL-safe, must be unique |
| description | ✅ | Short description (max 200 chars) |
| cuisine | ✅ | e.g., Indian, American, Italian |
| photo | ✅ | .jpg, .jpeg, .png, or .webp |
| prepTime | ✅ | Minutes (number) |
| cookTime | ✅ | Minutes (number) |
| servings | ✅ | Number of servings |
| difficulty | ✅ | Easy / Medium / Quick |
| category | ✅ | indian / breakfast / salads / soups / global / snacks |
| ingredients | ✅ | At least one item |
| steps | ✅ | At least one step |
| tags | ❌ | Optional array (quick, vegetarian, etc.) |
| isKidFriendly | ❌ | Optional boolean |
| isFeatured | ❌ | Optional boolean |

## Troubleshooting

### "Recipe with slug already exists"
```
Solution: Change slug to be unique
Example: "paneer-tikka" → "paneer-tikka-v2" or "paneer-tikka-smoky"
```

### "File not found" for photo
```
Solution: Use full absolute path
❌ Wrong: /photos/image.jpg
✅ Right: /Users/srishtigupta/Downloads/followthearoma/photos/image.jpg
```

### "Unsupported image format"
```
Supported: .jpg, .jpeg, .png, .webp
Use an image converter if needed
```

### Recipe created but not visible on site
```
Wait 30-60 seconds for Next.js ISR revalidation
Check: http://localhost:3000/recipes/your-slug
```
